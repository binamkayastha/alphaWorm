Class Overview

This class is used to render stokes and layers, to fill paths, and to perform rendering operations. It is the core of the WILL SDK Rasterizer module. It manages the allocated OpenGL resources and can only be used on an OpenGL thread.

Create an InkCanvas instance using one of the following factory methods:
InkCanvas.create()
InkCanvas.create(SurfaceHolder,EGLRenderingContext.EGLConfiguration)

You can only create a canvas on a valid OpenGL thread. If you create a canvas with InkCanvas.create(SurfaceHolder,EGLRenderingContext.EGLConfiguration), a proper EGLRenderingContext is created, applied, and dedicated to that canvas only, and the thread from which the method is called becomes the OpenGL thread.
If the current thread already owns a valid EGL context, create the canvas using InkCanvas.create(). In this case, a dedicated EGLRenderingContext is not created: the canvas uses the EGL context of the current thread.

The InkCanvas provides a group of layer creation methods. Use these to create and initialize Layer instances. A layer can only be used in the InkCanvas instance used to create it.

All rendering operations are performed on layers. Only one layer can be active at any given time. When you set a layer as active, using the InkCanvas.setTarget(Layer) method, it becomes the "target layer" of the canvas.

The InkCanvas class provides the InkCanvas.drawStroke(StrokePaint,StrokeJoin,FloatBuffer,int,int,int,float,float) method to render strokes. This method requires a StrokePaint instance, holding information about how to render the stroke, and a StrokeJoin instance "joining" two successive rendered strokes parts.

To draw a layer, call the InkCanvas.drawLayer(Layer,BlendMode) method.

Since an InkCavas instance can only be used on the thread from which it was initialized, you must release the InkCanvas instance's allocated OpenGL resources if the EGL context is going to be lost, or the rendering surface provided by the Android framework is going to be destroyed or recreated. The InkCanvas class implements the Disposable interface: call InkCanvas.dispose() for each canvas instance to finalize it properly and make it safe for garbage collection. When the InkCanvas.dispose() method is called, the InkCanvas instance automatically disposes all layers it created. To dispose a layer before its canvas is disposed, for example, to release resources, use its Layer.dispose() method.

Since: 1.2
