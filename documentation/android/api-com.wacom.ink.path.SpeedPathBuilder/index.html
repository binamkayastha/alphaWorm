public FloatBuffer beginPath(float x, float y, double timestamp)

Deprecated in 1.3. This method will be removed in the next version of the SDK. Use SpeedPathBuilder.addPoint(PathUtils.Phase,float,float,long) instead.

Starts building a path from velocity based input. Normally this method is called on a touch down event. The method calculates a partial path (a set of path control points) from the provided input. The returned partial path can be modified by clients before adding it to the currently built path with the PathBuilder.addPathPart(FloatBuffer,int) method (for example the part could be first smoothened).

Parameters

Name Description
x The x coordinate of the user's input in the desired path's coordinate system.
y The y coordinate of the user's input in the desired path's coordinate system.
timestamp The timestamp of the user's input in seconds with double precision. It is used for the velocity calculation.

Returns

A part of a path (a set of control points) as a float buffer.

Since: 1.2

public FloatBuffer addPoint(PathUtils.Phase phase, float x, float y, long timestamp)

This method adds a point to the path builder instance. Normally this method is called on a touch move event. Depending on the phase parameter, either a new path is started, a new point is added to a current built path, or the path is closed. A client can modify the returned partial path (for example, by smoothing it) before committing it to the path using the PathBuilder.addPathPart(FloatBuffer,int) method.

Parameters

Name Description
phase If Phase.BEGIN, a new path is started (if a path is already started, this method will fail); if Phase.MOVE, the current built path is continued (if the path is not started, this method will fail); if Phase.END, the path is closed (if the path is not started or already closed, this method will fail).
x The x coordinate of the user input in the desired path coordinate system
y The y coordinate of the user input in the desired path coordinate system
timestamp The timestamp of the user input in seconds with long precision. This is used to calculate velocity.

Returns

A partial path (a set of control points) as a float buffer

Since: 1.3
