Class Overview

The PathBuilder class and its two concrete subclasses, PressurePathBuilder and SpeedPathBuilder, translate user input into a geometric representation of a stroke.
PathBuilder converts series of data in (x, y, timestamp) or (x, y, pressure) format into series of spline control points in (x, y, width, alpha) format. These control points can be rendered as strokes, or used in other ways.
The PathBuilder class builds paths based on user input, input normalization, and preset path generation property configuration. The PathBuilder class normalizes the user input, then generates path properties based on that input. The result of the generation process is a list of control points with width and opacity as Catmull-Rom splines.
A pathbuilder instance includes configuration for normalization and properties. Normalization configuration specifies how the input will be normalized (clamped to the supported range). Property configuration specifies the PathBuilder properties that should be generated for each point. For example, to generate a width or an alpha property, you must configure it using the setPropertyConfig() method.
The PathBuilder class uses a two-step approach to allow transformations of an intermediate partial path (for example, by smoothing it) before committing it to the actual path.
The PathBuilder class allows you to obtain a preliminary path. This preliminary path can be used to reduce the perceived latency introduced by the interpolation method or any processing applied by the client.
WILL Core provides two different implementations of PathBuilder class:
PressurePathBuilder builds paths based on touch and pressure.
SpeedPathBuilder builds paths based on touch alone.

public FloatBuffer createPreliminaryPath()

This method creates a preliminary path for the current built path.

If stroke partial paths are smoothened using the MultiChannelSmoothener during stroke generation, there is a lag due to the smoothing algorithm. To neutralize this lag, the PathBuilder supports drawing preliminary stroke curves.

This method calculates a partial path (a set of control points) without updating the path builder state. This partial path behaves as though the path is finished. This part can be modified by a client before committing it to the preliminary path using the PathBuilder.finishPreliminaryPath(FloatBuffer,int) method. For example, it can be smoothened.

Returns

A partial path as a set of control points

Since: 1.2

public void setPropertyConfig(PathBuilder.PropertyName name, float minValue, float maxValue, float initialValue, float finalValue, PathBuilder.PropertyFunction function, float functionParameter, boolean bShouldFlip)

This method configures the PathBuilder to generate a specific property (width or opacity) based on the user input.

Parameters

Name Description
name The name of the property that will be configured
minValue The minimum property value
maxValue The maximum property value
initialValue The property value at the beginning of the path. If set to NaN, the value for the next point is taken.
finalValue The property value at the ending of the path. If set to NaN, the value for the previous point is taken.
function The function that converts the input into a property value. See PropertyFunction.
functionParameter The property function parameter
bShouldFlip Whether the property (width or opacity) changes in direct proportion (true) or inverse proportion (false) to increased user input. For example, if the path should be 5px wide when the user draws at a rate of 100 px per second and 10px wide when drawing with 200px per second, set bShouldFlip=1. Set bShouldFlip=0 to invert, producing a path 10px wide at 100px/sec and 5px wide at 200px/sec.

Since: 1.2
