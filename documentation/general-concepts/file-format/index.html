Overview

WILL file (.will) format facilitates the exchange of ink content between different applications. It is designed to accomodate multi-media content and therefore is suitable not only for pure inking applications, but also for applications that combine ink content with other media like images, text, shapes and etc. WILL file format is based on the following technologies:

  • WILL Encoding Schema - fast, compact, portable, binary encoding schema specifically designed for ink content. For more information check Encoding and decoding ink content.
  • Scalable Vector Graphics (SVG) - a language based on XML for describing two-dimensional vector and mixed vector / raster graphics. Suitable for combining multi-media content.
  • Open Packaging Convention - a container-file technology to store a combination of XML and non-XML files that together form a single entity.

It is important to note that RIFF-based container is deprecated and the new container specification for WILL file format is not backward compatible with the RIFF-based container.

WILL Metadata Specification

In addition to standard XMP schemes, WILL introduces a new one that allows metadata to be associated to a group of paths (strokes) instead of a whole document:

GroupList

Property Value Type Description
will:GroupList bag Group Contains a set of Groups each of them containing number of paths.

Group

Property Value Type Description
will:Elements bag Range Contains a set of Ranges each of them points to a sequential range of paths.
will:Extensions Undefined struct Any additional top-level XMP property to describe the content shown in image region.

Range Value Type

Field Name Value Type Description
will:from XPointer Element Expression pointing to the first path of the range.
will:to XPointer Element Expression pointing to the last path of the range.

XPointer Element Value Type W3C XPointer Element for addressing elements in documents with hierarchical object model.

XMP chunk is created according to recommendation of XMP specification. Below you can find an example of WILL scheme:

<?xpacket begin="" id="W5M0MpCehiHzreSzNTczkc9d"?>
...
<rdf:Description rdf:about="" xmlns:will="http://www.wacom.com/will/meta/groups/" xmlns:dc="http://purl.org/dc/elements/1.1/">
  <will:GroupList>
    <rdf:Bag>
      <rdf:li rdf:parseType="Resource">
        <will:Elements>
          <rdf:Bag>
            <rdf:li rdf:parseType="Resource">
              <will:Range will:from="element(/1/1)" will:to="element(/1/50)" />
            </rdf:li>
...
          </rdf:Bag>
        </will:Elements>
        <will:Extensions rdf:parseType="Resource">
          <dc:creator>
            <rdf:Seq>
              <rdf:li>David Smith</rdf:li>
            </rdf:Seq>
          </dc:creator>
        </will:Extensions>
      </rdf:li>
...
    </rdf:Bag>
  </will:GroupList>
</rdf:Description>
...
<?xpacket end="w"?>
